# web-main-guide

## 项目简介

`web-main-guide` 是一个用于归档和发布多个独立 HTML 页面的仓库。这里存放的是各种静态网页资源，可能包括工具页面、信息展示页、导航页等，不针对任何特定域名或具体网站。

本仓库的目的是提供一个整洁、可维护的静态页面集合，方便通过 GitHub Pages 或其他静态托管方式直接访问。

## 目录说明

仓库根目录下主要包含以下内容：

- `index.html` — 仓库入口页面，通常作为导航或总览。
- `pages/` — 存放各个独立 HTML 页面的子目录，每个页面一个文件夹，便于管理。
- `assets/` — 公共资源目录，包括样式表（CSS）、脚本（JavaScript）、图片等。
- `docs/` — 可选，用于存放额外的文档或说明文件。

目录结构示例（仅供参考）：

```
web-main-guide/
├── index.html
├── pages/
│   ├── page-a/
│   │   └── index.html
│   └── page-b/
│       └── index.html
├── assets/
│   ├── css/
│   ├── js/
│   └── images/
└── README.md
```

## 页面归档说明

每个独立页面应遵循以下归档规范：

- 每个页面存放于 `pages/` 下的独立文件夹中，文件夹名称使用英文小写、短横线分隔。
- 页面入口文件统一命名为 `index.html`。
- 页面所需的资源（CSS、JS、图片等）优先放在该页面的子文件夹内，或引用 `assets/` 下的公共资源。
- 页面之间不应存在硬编码的相互依赖，保证每个页面可独立访问。

## 维护说明

### 添加新页面

1. 在 `pages/` 下创建新文件夹。
2. 将 HTML 文件及相关资源放入文件夹。
3. 如需要，更新 `index.html` 中的导航链接。

### 更新现有页面

直接修改对应文件夹中的文件，注意保持与其他页面的兼容性。

### 删除页面

移除对应文件夹，并清理 `index.html` 中的相关链接。

### 注意事项

- 请勿在页面中引入外部不可控资源，以免影响访问稳定性。
- 保持代码简洁，避免冗余和过大的文件。
- 建议为每个页面添加基本的响应式支持。
